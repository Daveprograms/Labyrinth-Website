Enhanced Introduction to Java

Brief History of Java and Its Evolution

Java was developed by James Gosling at Sun Microsystems and released in 1995. Originally called Oak, it was designed for interactive television, but was too advanced for the digital cable television industry at the time. Java's "Write Once, Run Anywhere" (WORA) principle has made it one of the most popular programming languages.

Overview of the JDK, JRE, and JVM

  • JDK (Java Development Kit): A software development kit needed to develop Java applications. It includes the JRE, a compiler, and other tools.
  • JRE (Java Runtime Environment): Provides the libraries and the JVM to run applications written in Java.
  • JVM (Java Virtual Machine): A runtime engine that executes Java bytecode. It enables Java's WORA principle by being available on many platforms.

Explanation of Java's WORA Principle

Java's "Write Once, Run Anywhere" principle allows compiled Java bytecode to run on all platforms without needing recompilation, thanks to the JVM.

Basics of Java Programming

Detailed Syntax Explanations

  • Variables: Reserved memory locations to store values. Defined by data types such as int, double, boolean.
  • Data Types: Include primitive (e.g., int, double) and reference/object types (e.g., String).
  • Operators: Includes arithmetic (+, -), relational (==, !=), and logical (&&, ||) operators.
  • Control Flow Statements: if-else statements, for, while, and do-while loops for decision making and repetitive tasks.

Explanation of Java Packages and Import Statements

Packages organize classes into a folder structure for easy management. import statements allow your class to use classes from other packages.

Detailed Section on Java Methods, Including Method Overloading

Methods in Java perform specific tasks and improve code reusability. Method overloading allows multiple methods with the same name but different parameter lists, enabling polymorphism.

Real-life Example of Method Overloading

class MathOperation {
    public int add(int a, int b) { return a + b; }
    public int add(int a, int b, int c) { return a + b + c; }
    public double add(double a, double b) { return a + b; }
}

public class Test {
    public static void main(String[] args) {
        MathOperation operation = new MathOperation();
        System.out.println(operation.add(10, 20)); // Outputs 30
        System.out.println(operation.add(10, 20, 30)); // Outputs 60
        System.out.println(operation.add(10.0, 20.0)); // Outputs 30.0
    }

What is Java?

Java is an object-oriented, class-based programming language designed to have as few implementation dependencies as possible. It allows developers to "write once, run anywhere" (WORA), meaning that Java code compiled on one platform does not need to be recompiled to run on another. Java is widely used for developing application software and deploying it in cross-platform computing environments.

How Java Differs from Other Programming Languages

  • Platform Independence: Java provides a unique feature of not being tied to a specific hardware or operating system. This is achieved through the use of the Java Virtual Machine (JVM).
  • Object-Oriented: Everything in Java is an object, which encapsulates data and operations on data, making it easier to structure and manage complex programs.
  • Robust and Secure: Java offers strong memory management and protection against many vulnerabilities, such as buffer overflows. It also includes a security manager that defines the access rules for classes.
The Significance of the public static void main(String[] args) Method

The public static void main(String[] args) method is the entry point for a Java application. When the Java Virtual Machine (JVM) starts, it looks for this specific method signature to begin execution of the Java program.

  • public: The method is accessible from any other class.
  • static: The method can be called without creating an instance of the class containing the main method.
  • void: The method does not return any value.
  • main: The name of the method searched for by the JVM as the starting point of the application.
  • String[] args: The method accepts a single argument: an array of strings, which allows command-line arguments to be passed to the application.

Polymorphism

Just as a person can have different roles such as a mother, a writer, or a student, in programming, polymorphism allows objects to take on many forms. A method in a parent class can be overridden in its subclass to perform different operations.

class Person {
    void role() {
        System.out.println("Person has multiple roles.");
    }
}
class Mother extends Person {
    void role() {
        System.out.println("Person is a Mother now.");
    }
}
class Writer extends Person {
    void role() {
        System.out.println("Person is a Writer.");
    }
}
class Student extends Person {
    void role() {
        System.out.println("Person is a Student.");
    }
}
public class Main {
    public static void main(String[] args) {
        Person p = new Mother();
        p.role(); // Outputs "Person is a Mother now."
        p = new Writer();
        p.role(); // Outputs "Person is a Writer."
        p = new Student();
        p.role(); // Outputs "Person is a Student."
    }
}

Encapsulation

A company has different departments like Production, HR, and Marketing, each encapsulating specific functions. Similarly, encapsulation in programming groups together data and methods that manipulate the data, keeping it safe from outside interference and misuse.

class Company {
    private String department;
    private int employeeCount;
    public String getDepartment() {
        return department;
    }
    public void setDepartment(String dept) {
        this.department = dept;
    }
    public int getEmployeeCount() {
        return employeeCount;
    }

        this.employeeCount = count;

    }
}

WHAT IS POLYMORPHISM? WHAT MANIFESTATIONS OF POLYMORPHISM IN JAVA DO YOU KNOW?
Polymorphism (from the Greek polymorphous) is a property that allows the same name to be used to solve two or more similar but technically different tasks. The goal of polymorphism, as applied to object-oriented programming, is to use a single name to define common actions for a class. The execution of each specific action will be determined by the data type. More generally, the concept of polymorphism is the idea of "one interface, many methods". This means that you can create a common interface for a group of similar actions. For example, you have an acceleration pedal on an electric car and on a car with a gasoline engine. In both cases, pressing the acceleration pedal would bring us the same result - the car will go faster. But the way how exactly acceleration will be implemented in electric and non-electric cars - will be different because of the different car construction.
WHAT IS ENCAPSULATION?
Encapsulation is hiding the implementation of a class and separating its internal representation from the external (interface). This is a property that allows you to close access to the fields and methods of the class to other classes, and provide them with access only through the external interface (methods). When using an object-oriented approach, it is not ok to directly access the properties of a class from the methods of other classes. To access the properties of a class, usually, you need to use special methods of this class to get and change its properties.
WHAT ARE THE ADVANTAGES OF OBJECT-ORIENTED PROGRAMMING LANGUAGES?
  1. Modularity for easier troubleshooting
    When working with object-oriented programming languages, you know exactly where to look when something goes wrong. This modularity allows an IT team to work on multiple objects simultaneously while minimizing the chance of duplication.
  2. Reuse of code through inheritance
    The inheritance technique saves time by allowing the creation of a generic class (e.g., Car), and then subclasses (e.g., RaceCar and Limousine) that inherit the generic class’s traits. This enables code reuse and simplifies maintenance.
  3. Flexibility through polymorphism
    Polymorphism allows a single function to adapt to whichever class it’s in, enabling a common interface for a group of actions. This flexibility aids in the creation of more dynamic and versatile code.
  4. Effective problem solving
    Object-oriented programming enables breaking down a large problem into solvable chunks, making it a more intuitive approach for developing large projects. The reuse of classes further accelerates the development process.

THERE IS THE EXPRESSION "IS" AND "HAS". WHAT DO THEY MEAN IN TERMS OF OOP PRINCIPLES? WHAT IS THE DIFFERENCE BETWEEN COMPOSITION AND AGGREGATION?

"is" represents inheritance, whereas "has" represents composition in object-oriented programming. For example, a House "is" a Building (inheritance), but a House "has" a Bathroom (composition). If there is no "is" relationship, it's better to use composition to reuse code. Interfaces should be used for polymorphism when there's no "is" key dependency.

An association denotes a relationship between objects. Aggregation and composition are special cases of association. Aggregation implies a "part-of" relationship, but components in an aggregation can belong to multiple aggregations. Composition is a stricter form of aggregation where a part cannot belong to multiple compositions and is destroyed when the composite is.

For example, a motorcycle "has" a luggage bag (association), a motorcycle "is composed of" wheels (composition), and a person "is part of" an interest group (aggregation), but may belong to several groups.

WHAT DO YOU MEAN BY POLYMORPHISM, ENCAPSULATION, AND DYNAMIC BINDING?

Polymorphism refers to the ability of a variable of a given type to refer to objects of different types by calling a method that is specific to the particular type of object reference. The advantage of polymorphism is that it allows you to add new derived object classes without breaking the calling code, a concept also known as dynamic object binding.

Consider an example of polymorphism with classes: Shape, Circle, and Triangle, where Circle and Triangle inherit from Shape. Each class has a "draw" method, which is overridden in Circle and Triangle. When we create an object of type "Shape" and assign it a reference to an object of the "Circle" type, calling the "draw" method on this object results in the method of the "Circle" class being invoked.

Example: Figure f = new Circle(); f.draw();

This encapsulation of behavior can be further utilized with interfaces, allowing for dynamic binding, where the specific method to be invoked at runtime depends on the object behind the reference.

DESCRIBE ACCESS MODIFIERS IN JAVA.

Java has the following access modifiers:

  • private: Used in constructors, inner classes, methods, and class fields - Access is allowed only within the current class.
  • default (package-private): Used in classes, constructors, interfaces, inner classes, methods, and class fields - Package level access. A class declared this way will only be available within its package.
  • protected: Used in constructors, inner classes, methods, and class fields - Provides access within the package and also to subclasses outside the package.
  • public: Used in classes, constructors, interfaces, inner classes, methods, and class fields - Makes the member accessible from any other class.

The sequence of modifiers in descending order of privacy level is: private, default, protected, public.

HOW DOES AN ABSTRACT CLASS DIFFER FROM AN INTERFACE? WHEN WOULD YOU USE AN ABSTRACT CLASS AND WHEN WOULD YOU USE THE INTERFACE?
An abstract class is a class that is marked "abstract" and may or may not contain abstract methods. An instance of an abstract class cannot be instantiated. A class that inherits from an abstract class may or may not implement abstract methods. In case a child class doesn’t implement all abstract methods, then it must also be abstract. Also, if the inheritor class overrides the method implemented in the abstract parent class, it can be overridden with the abstract modifier! That is, to abandon the implementation. Accordingly, this class must also be abstract as well.

As for the interface, it contains only abstract methods and constants, this was the case before the release of Java 8. Starting with Java 8, in addition to abstract methods, we can also use standard methods (default methods) and static methods (static methods) in interfaces.
A Default method in an interface is a method in an interface with default logic that is not required to be defined in the implementation of that interface.
Static methods in an interface are essentially the same as static methods in an abstract class.
When implementing an interface, a class must implement all methods of the interface. Otherwise, the class must be marked as abstract. An interface can also contain inner classes. And no abstract methods in them.
Also, always remember that you can extend only one class in Java, but you can implement multiple interfaces.
What to use: Interface or Abstract class?
An abstract class is used when we need some kind of default implementation. An interface is used when a class needs to specify specific behavior. Often an interface and an abstract class are combined, i.e., implement an interface in an abstract class to specify the default behavior and implementation.

Type casting is setting the type of a variable or object to something other than the current type. There are two types of cast in Java: automatic and not automatic. Automatic happens, for example, in such cases: byte->short->int->long->float->double, that is, if we extend the type, then no explicit conversion is required, the cast happens automatically. If we are narrowing, then it is necessary to explicitly specify the type conversion. In the case of objects, we can automatically cast fromthe child type to the parent, but not vice versa, then a ClassCastException will be thrown.
WHAT IS A STATIC CLASS, WHAT ARE THE FEATURES OF ITS USE?
A static class can only be a nested class (a class definition placed inside another class). Unlike an ordinary inner class that holds a reference to an object of the outer class, a static inner class does not. This means an object of the outer class is not needed to create an object of a static inner class, and from an object of a static nested class, you cannot directly access the non-static members of the outer class. Additionally, ordinary inner classes cannot contain static methods and members.

Nested classes are useful because each inner class can independently inherit a specific implementation, providing a solution to the problem of multiple inheritance by allowing the nested class to inherit from another class while the outer class might already inherit from a different class.
WHAT TYPES OF INNER CLASSES ARE THERE? WHAT ARE THEY USED FOR?
Nested classes, available since Java 1.1, can be of four types:
  • Static nested classes: Have access to any static methods of its outer class, including private ones. It cannot access non-static fields and methods directly, only through a reference to an instance of the parent's class.
  • Non-static inner classes: Declared inside another class and have access to the non-static properties of the outer parent class.
  • Local classes: Defined in a block, such as within the body of a method, and have scope limited to that block.
  • Anonymous classes: Do not have a name and are declared and instantiated all at once, typically used for implementing interface methods or extending classes on the fly.
Each type serves different purposes, such as accessing specific members of the outer class, encapsulating helper classes, defining lightweight classes for one-time use, and more.
IS IT POSSIBLE TO CHANGE WHEN OVERRIDE THE METHOD: Access Modifier, return type, Argument type or number of arguments, Argument name, Change the order, quantity, or remove the throws section altogether?
When overriding a method, the following changes are permissible:
  • Access Modifier: Yes, if you expand access (package -> protected -> public).
  • Return Type: Yes, if downcasting is performed. The return type in the overridden method of the heir class must NOT be wider than in the parent class (Object -> Number -> Integer).
  • Argument Type or Number of Arguments: No, changing these leads to method overloading, not overriding.
  • Argument Name: Yes, argument names can be changed as they do not affect polymorphism.
  • Throws Section: It is possible to change the order, remove the throws section altogether, or add new exceptions that inherit from declared or runtime exceptions.
Important points on method overriding:
  • The access modifier in the method of the descendant class must NOT be narrower than in the parent class.
  • The description of the exception in the overridden method of the descendant class must NOT be wider than in the parent class.
  • A method declared as "private" in a parent class cannot be overridden.
Method overriding is valid when classes are inherited, ensuring the method has the same signature in the descendant class as in the parent class.
WHAT IS A DEFAULT CONSTRUCTOR?
In Java, a default constructor is provided by the compiler if no constructors are explicitly defined in a class. This constructor is a no-argument constructor that does not perform any special actions or initializations. It simply allows for the creation of an object. If a class does not declare any constructors, the Java compiler automatically provides a default constructor. However, if any constructors are explicitly declared (whether they take parameters or not), the compiler does not generate a default constructor. Additionally, if the derived class does not explicitly call the base class constructor using super(), the default constructor of the base class is implicitly called. If the base class lacks a default constructor, it results in a compilation error.

Java Collection Framework

WHAT IS A COLLECTION IN JAVA COLLECTION FRAMEWORK?

Collections in the Java Collection Framework are data structures that primarily serve to store and manage objects. They provide an efficient way to access, modify, and manipulate collections of objects through a set of well-defined interfaces and implementations. This framework supports operations such as adding a new element, removing an element, and altering an element within the collection.

NAME THE MAIN INTERFACES OF COLLECTIONS AND THEIR IMPLEMENTATIONS.

The Java Collection Framework defines several core interfaces, each with multiple implementations:

  • List: An ordered collection that may contain duplicate entries. Common implementations include ArrayList, LinkedList, and Vector.
    • ArrayList is optimized for random access of elements.
    • LinkedList excels in adding and removing elements.
    • Vector is similar to ArrayList but is synchronized for thread safety.
  • Set: A collection that cannot contain duplicate elements. Implementations include HashSet, TreeSet, and LinkedHashSet.
    • HashSet stores its elements in a hash table, is best for rapid lookups.
    • TreeSet stores elements in a red-black tree, maintaining order.
    • LinkedHashSet maintains insertion order.
  • Queue: Designed for holding elements prior to processing. Main implementations are LinkedList and PriorityQueue.
    • LinkedList can also implement a dequeue.
    • PriorityQueue orders elements according to their natural ordering or by a Comparator provided at queue construction time.
  • Map: An object that maps keys to values, with no duplicate keys allowed. Notable implementations include HashMap, TreeMap, and LinkedHashMap.
    • HashMap allows null values and the null key.
    • TreeMap is sorted according to the natural ordering of its keys or by a Comparator provided at map creation time.
    • LinkedHashMap maintains insertion order.

It's also worth noting that collections and maps can be made thread-safe using the Collections.synchronizedMap and Collections.synchronizedList methods.
